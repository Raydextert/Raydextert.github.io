线程基础

由于下文所用的方法代码都在一个都已封装在一个类型，故先在这里进行说明

  //静态类
public static class Comman
{
    public static void PrintNumbers()
    {
        Console.WriteLine("开始打印数字...");
        for (int i = 0; i < 10; i++)
        {   
            //格式化字符串中前面的数字，如0、1可以表示为后面要格式化输出对象的索引

            Console.WriteLine(string.Format("当前线程ID：{0:G},当前数字：{1:G}",Thread.CurrentThread.ManagedThreadId,i));
        }
    }

    public static void PrintNumbersWithDelay()
    {
        Console.WriteLine("开始打印数字...");
        for (int i = 0; i < 10; i++)
        {
            // 暂停线程
            Thread.Sleep(1000);
            Console.WriteLine(string.Format("当前线程ID：{0:G},当前数字：{1:G}", Thread.CurrentThread.ManagedThreadId, i));
        }
    }

    public static void PrintNumbersWithStatus()
    {
        Console.WriteLine("开始打印数字...");
        string threadState = Thread.CurrentThread.ThreadState.ToString();

        Console.WriteLine(string.Format("当前线程的状态：{0:G}", threadState));

        for (int i = 0; i < 10; i++)
        {

            Thread.Sleep(1000);
            Console.WriteLine(string.Format("当前线程ID：{0:G},当前数字：{1:G}", Thread.CurrentThread.ManagedThreadId, i));
        }
    }

}
 ### 1、线程暂停      线程暂停指的是调用Thread静态方法Sleep方法使得当前线程在指定时间内暂停执行代码，但是这里需要注意的是子线程在主线程中的启动位置。如下面两段代码所示： 
  一、子线程在主线程调用方法后启动
  //主线程 
  Comman.PrintNumbersWithDelay();
  //支线程
  Thread thread = new Thread(Comman.PrintNumbers);
  thread.Start();


  //暂停窗口
  Console.ReadKey();

============================
二、子线程在主线程调用方法前面启动
 //支线程
  Thread thread = new Thread(Comman.PrintNumbers);
  thread.Start();  
  //主线程 
  Comman.PrintNumbersWithDelay();
 支线程在主线程的方法前启动（代码二），则主线程和子线程都将开始执行各自的代码块，而在如果在后面（代码一）则会先调用方法待方法执行完之后，再执行支线程,因为这时支线程还没有开始启动

2、线程等待

     这里调用的是Thread的Thread.join()方法，目的是阻塞其他线程的执行直到当前线程执行完成

3、线程终止

     可以调用Thread中的Thread.abort()的方法，原理是向正在执行的线程扔出一个异常，从而使得线程终止。但是这方法并不值得推崇。因为，该异常可以在任何时刻发生并可能彻底摧毁应用程序。并且，使用这种方法也不一定总能终止该线程。目标线程可以通过处理该异常的并调用Thread.ResetAbort()方法来拒绝被终止

4、线程优先级

     这里分两种情况
     1、当多核运行多线程时：多线程不管优先级如何都有概率被CPU调用。优先级的高低只是决定了线程被CPU执行的概率大小，并不决定线程的执行顺序
     2、当单核运行多线程时，可以理解为CPU只有一双手，所以同一时间只能执行一个线程，因此线程的优先级高低一般与执行顺序有关，通常一般是执行线程优先级更高的线程，但是并不一定要等到执行完高优先级的线程之后，再来执行低优先级的线程，同时也拥有概率执行低优先级的线程

5、线程的状态

     通过查看Thread.ThreadState属性即可知道

6、前后台线程

     1、前台线程：前台线程是不会被立即关闭的，它的关闭只会发生在自己执行完成时，不受外在因素的影响。假如应用程序退出，造成它的前台线程终止，此时CLR仍然保持活动并运行，使应用程序能继续运行，当它的的前台线程都终止后，整个进程才会被销毁
     2、后台线程是可以随时被CLR关闭而不引发异常的，也就是说当后台线程被关闭时，资源的回收是立即的，不等待的，也不考虑后台线程是否执行完成，就算是正在执行中也立即被终止。
二者的区别：应用程序必须完成所有前台线程的才会完全退出，若前台线程未执行完成，关闭应用程序之后应用程序并未完全退出，任务管理器中还存在此线程。而对于后台线程，应用程序则可以不考虑其是否已经运行完毕而直接退出，即应用程序结束时，所有后台线程都会自动结束。

7、向线程传递参数

其实是将方法传递给线程，只是这个方法使用lambda表达式表示的

//使用lambda表达式引用另一个C#对象称为闭包,本质上还是生成一个委托
 Thread t3 = new Thread(() => { CountNumbers(9); });
 t3.Name = "t3";
 t3.Start();
 t3.Join();
 Console.WriteLine("=========================================");

 var t4 = new Thread(() => { PrintNumbers(i); });
 t4.Name = "t4";
 i = 12;

 t5.Name = "t5";

 Console.ReadKey();

a标签引入脚本以及layer插件的使用

Posted on 2019-11-28

关于html中a标签传入脚本的几种方式

1、直接在href属性中添加方法形如：< a href=”javascript:js_method()”>javascript只是一个协议，后面紧跟着的是一个调用方法，注意调用的是方法，而不是从外部引入的脚本文件。
2、在a标签中的onclick事件中绑定方法，形如：< a href=”javascript:void(0)” onclick=”method()”>void是一个操作符，void(0)则表示的是地址不发生跳转。
3、形如< a href=”#” onclick=”js_method();return false;”>执行了js之后页面不发生跳转
注：以上的js都指的是的方法而不是js文件
